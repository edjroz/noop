# `Tools/defradb-embed/`

Build pipeline that turns the Phase 1 Go module at
`Packages/DefraSync/defradb-embed/` into a Swift-linkable
`DefraEmbed.xcframework` Strand can consume.

## Quick start

```bash
./Tools/defradb-embed/build.sh
```

Builds for both `darwin/arm64` and `darwin/amd64`, lipos them into one
universal binary, and packages the result at
`Packages/DefraSync/DefraEmbed.xcframework/` (gitignored).

First build is slow (~10 min — Go has to compile the full DefraDB transitive
dep tree, ~3600 modules). Subsequent builds are 1–2 min courtesy of Go's
build cache.

## Files

- `build.sh` — the pipeline. Per-arch `go build -buildmode=c-shared` → `lipo`
  → hand-rolled `.framework` → `xcodebuild -create-xcframework`.
- `module.modulemap.template` — drops into
  `DefraEmbed.framework/Modules/module.modulemap` to make the framework
  Swift-importable as `import DefraEmbed`.

## Output layout

```
Packages/DefraSync/DefraEmbed.xcframework/
├── Info.plist                          (generated by xcodebuild)
└── macos-arm64_x86_64/
    └── DefraEmbed.framework/
        ├── DefraEmbed                  → Versions/A/DefraEmbed (universal dylib)
        ├── Headers/                    → Versions/A/Headers/   (DefraEmbed.h)
        ├── Modules/                    → Versions/A/Modules/   (module.modulemap)
        ├── Resources/                  → Versions/A/Resources/ (Info.plist)
        └── Versions/{A,Current}/...
```

The dylib's install name is set to `@rpath/DefraEmbed.framework/DefraEmbed`, so
the Strand app can embed the framework anywhere on `@rpath` without recording
absolute paths.

## Exported C ABI

All names prefixed `Defra…` to avoid C symbol collisions in the host process:

| Symbol | Purpose |
|---|---|
| `DefraStartNode(rootdir, httpListen, p2pListen, devMode) -> *char` | Bring up the in-process DefraDB node. Returns nil on success, error string otherwise. |
| `DefraStopNode() -> *char` | Tear it down. Idempotent. |
| `DefraAPIBaseURL() -> *char` | Returns `http://…:9181` or nil if not running. |
| `DefraLoadCollections(sdl) -> *char` | Schema bootstrap. Idempotent. |
| `DefraP2PSubscribe(namesJSON) -> *char` | Subscribe to pubsub topics. |
| `DefraP2PConnect(multiaddr) -> *char` | libp2p dial. |
| `DefraP2PReplicateTo(multiaddr, namesJSON) -> *char` | Forward-push replicator. |
| `DefraSelfMultiaddrsJSON(**result, **errOut)` | Returns JSON array of multiaddrs. |
| `DefraActivePeersJSON(**result, **errOut)` | Returns JSON array of peers. |
| `DefraFreeString(s)` | Releases any C string returned above. **Mandatory** — Swift must `defer DefraFreeString(...)` after consuming each pointer. |

Phase 3 will write a thin Swift wrapper that hides the manual memory
management; for Phase 2 we ship the framework only.

## Verify

```bash
# Universal? Both archs should be listed.
lipo -info Packages/DefraSync/DefraEmbed.xcframework/macos-arm64_x86_64/DefraEmbed.framework/DefraEmbed

# All Defra symbols present?
nm -gU Packages/DefraSync/DefraEmbed.xcframework/macos-arm64_x86_64/DefraEmbed.framework/DefraEmbed | grep _Defra

# Stripped size? Expect ~60–110 MB.
du -sh Packages/DefraSync/DefraEmbed.xcframework
```

## What this DOESN'T do

- iOS slices — Phase 4 problem. Add `ios-arm64`, `ios-simulator-arm64`, and
  `ios-simulator-x86_64` then.
- Code-signing / notarization — local dev runs only.
- Strand integration / `Package.swift` `.binaryTarget` — Phase 3.
- CI build — local-only for now.
